Development Diary – 05/03/2026
Task / Work Summary

This session focused on retraining the deepfake detection model with enhanced performance tracking, fine-tuning, and explainability integration.
I also developed a functional and accessible front-end interface to make the deepfake analysis process user-friendly, visually clear, and aligned with accessibility standards.

Development Diary Table
Date – [05/03/2026]
Context / Problem / Challenge

The previous version of my deepfake detector achieved moderate accuracy (~71%) but lacked deeper explainability and consistent validation performance, so set up for this implementation needed to be made visually.
Additionally, the system lacked a visual interface to allow users to upload videos and view detection results interactively.
The challenge was to refine the training pipeline for better stability and create a usable front-end that makes the model outputs interpretable and accessible.

Approach / Method

To improve training:

I refactored the model generation script to include a fine-tuning stage, data augmentation, L2 regularization, and dropout.

I implemented two training phases — a frozen-base stage and a fine-tuning stage for the top Xception layers.

A model evaluation section was added to generate a confusion matrix, classification report, and accuracy/loss graphs automatically after training.

Training metrics and plots were saved for traceability and further analysis.

For the UI:

I built a front-end interface that lets users upload or paste a YouTube link, triggering automatic video download, frame extraction, and analysis.

The results are displayed in a clean, color-coded format, with intention to include explainable factors such as “Facial Warping,” “Lighting Inconsistencies,” and “Texture Mismatches.”

Accessibility considerations included high-contrast color schemes, and clear status indicators.

Response / Solution

The retrained model achieved ~71.5% validation accuracy, maintaining consistency between training and validation (indicating low overfitting).

A full model performance report (JSON + PNG plots) is generated after each run, aiding transparent reporting and later sprint comparisons.

The front-end dashboard successfully connects to the backend analysis pipeline and allows users to view Grad-CAM results, prediction confidence, and factor explanations interactively.

The workflow is now modular and explainability-ready, with visual interpretability through Grad-CAM heatmaps ready to be implemented.

# Fine-tuned Deepfake Detector Model
base_model = Xception(weights='imagenet', include_top=False, input_shape=(150, 150, 3))
base_model.trainable = False

model = models.Sequential([
    base_model,
    layers.GlobalAveragePooling2D(),
    layers.Dense(128, activation='relu', kernel_regularizer=tf.keras.regularizers.l2(0.001)),
    layers.Dropout(0.5),
    layers.Dense(1, activation='sigmoid')
])

model.compile(
    optimizer=optimizers.Adam(learning_rate=1e-4),
    loss='binary_crossentropy',
    metrics=['accuracy']
)
