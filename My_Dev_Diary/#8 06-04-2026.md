Development Diary — Transition to Hybrid Static + Video Deepfake Training Strategy

Phase: Dataset Design / Implementation Planning
Date: 6 April 2026

Context / Problem / Challenge

The initial version of the deepfake detection system was trained using a static image dataset consisting of individual face images labelled as real or fake. This dataset allowed the model to learn spatial manipulation artifacts, such as texture inconsistencies, blending errors, unnatural facial boundaries, and warping effects. These features are important for detecting deepfakes in single images and provide a strong baseline for spatial analysis.

However, deepfakes are most commonly distributed as videos, where manipulation artifacts may only become visible across consecutive frames. These include flickering, inconsistent lighting, and unstable facial alignment. A model trained purely on static images cannot effectively capture these temporal inconsistencies, which created a limitation relative to the overall project aim of developing a robust, real-world deepfake detection system.

To address this, I introduced video-derived training data using the FaceForensics++ (C23) dataset. This dataset contains approximately 7,000 manipulated videos generated using multiple deepfake techniques and 1,000 real videos. Frames extracted from these videos allow the model to learn artifacts present in dynamic content.

Rather than replacing the original static image dataset, I decided to adopt a hybrid training strategy. The original image dataset continues to serve an important role in teaching the model spatial manipulation features, while the video-derived frames are used to expose the model to temporal inconsistencies. This combined approach supports the development of a detector capable of analyzing both individual frames and video sequences.

Approach / Method

To construct the video-based dataset efficiently, I evaluated whether to extract frames from all available videos. Extracting frames from all 7,000 manipulated videos and 1,000 real videos would significantly increase preprocessing time and slow experimentation. Therefore, I opted to sample a subset of videos from each manipulation category while maintaining representation across all deepfake techniques.

The FaceForensics++ dataset is organized into folders corresponding to different manipulation methods (Deepfakes, FaceSwap, FaceShifter, NeuralTextures, Face2Face, and DeepFakeDetection). I initially implemented a cap of 200 videos per category to ensure balanced coverage across manipulation techniques while limiting extraction time.

However, I identified an issue with selecting the first 200 videos from each folder. The dataset uses aligned filenames, meaning each manipulation folder contains different deepfake versions of the same original video identities. Selecting the first 200 entries therefore resulted in repeated identities across manipulation types. This reduced dataset diversity and risked the model learning identity-specific features rather than general manipulation artifacts.

To improve diversity, I modified the sampling strategy to randomize video selection before applying the cap. This ensured that different identities were selected for each manipulation technique.

Response / Solution

I updated the extraction logic to shuffle the video list within each folder before selecting the capped subset. This approach increases variability across identities and prevents repeated samples derived from the same original videos. The implemented logic was:

random.shuffle(videos)
videos = videos[:MAX_PER_CATEGORY]

This modification improved dataset quality by:

Increasing identity diversity across manipulation types
Reducing bias toward specific individuals
Preventing overfitting to repeated faces
Encouraging the model to learn manipulation artifacts
Improving generalization to unseen videos

The dataset strategy now combines:

Original static image dataset → teaches spatial deepfake artifacts (texture, blending, warping)
Video-derived frame dataset → teaches temporal inconsistencies (flickering, lighting changes, motion instability)

This hybrid approach allows the model to leverage both spatial and temporal information, supporting the development of a more robust deepfake detector.

Outcome / Reflection

The transition to a hybrid dataset significantly strengthens the training pipeline. The original static image dataset remains valuable for learning fine-grained spatial inconsistencies, while the video-derived frames introduce temporal variation that better reflects real-world deepfake content. Randomizing the sampled videos further improves dataset diversity and reduces the risk of overfitting.

This work aligns directly with the broader project goal of developing an explainable deepfake detection system capable of analyzing video content. By combining spatial and temporal training data, the model is better positioned to detect both single-frame manipulations and inconsistencies that occur over time.
