# Development Diary - 23/12/2025

## Task / Work Summary
Prepared the deepfake project for Google Cloud deployment: made the backend cloud-ready (cross-platform ffmpeg, CORS, health check, 720p constraint, env-config host/port), updated the Chrome extension to use configurable endpoints, and added deployment scripts/guides (GOOGLE_CLOUD_SETUP.md, setup_vm.sh, NEXT_STEPS.md, refreshed CHECKLIST.md).

## Context / Problem / Challenge
Sciama HPC access is still pending/uncertain, and I need a reliable GPU environment for end-to-end video analysis (deepfake detection, Whisper, Wav2Lip). I must have a deployable API that the browser extension can call, without waiting for HPC queue/admin delays.

## Approach / Method
- Chose Google Cloud GPU VM (T4, preemptible) with clear budgeting and start/stop control; prepared all code to be Linux-compatible.
- Refactored backend:
  - Cross-platform ffmpeg detection (env var -> PATH -> Windows fallback -> /usr/bin/ffmpeg).
  - Added CORS middleware, health check `/`, 720p download cap, cleanup of old downloads, better errors.
  - Env-based host/port for uvicorn.
- Extension updates: configurable API_ENDPOINT, host_permissions widened, clearer loading message.
- Authored setup docs: full VM guide, automated setup script, roadmap, and refreshed checklist.
- Cost planning: preemptible T4 ~ $0.22/hr → ~450 hrs (25 hr/week) ≈ $285 total, within existing credits.

## Response / Solution
- Codebase now runs on Windows locally and will run on Ubuntu GPU VM without edits.
- Extension can flip from localhost to cloud by changing one line (API_ENDPOINT) and host_permissions.
- Deployment is scripted (setup_vm.sh, systemd optional) with a detailed GOOGLE_CLOUD_SETUP.md.
- Budget risk mitigated via preemptible instances and strict start/stop routine.

## Issues / Fixes
- Hardcoded Windows ffmpeg path → fixed with auto-detection and optional FFMPEG_PATH env var.
- Extension locked to localhost → fixed with configurable endpoint and broader host permissions.
- Potential CORS blocks → added permissive CORS (can tighten later to extension ID).
- Disk bloat from multiple downloads → cleanup of old mp4s before each run.
- Unclear deployment steps → added end-to-end guide and VM setup script.

## Why Google Cloud over Sciama HPC
- Immediate availability and simpler networking for the browser extension (public IP/port 8000).
- Preemptible T4 keeps costs within credits; clear hourly control via start/stop.
- Avoids queue/admin delays on Sciama; no SLURM job wrapping needed for the API.
- Easier to bind FastAPI/uvicorn to 0.0.0.0 and expose to the extension.
