# Development Diary — UI Consolidation, Forensic Pipeline Stabilization & Download Robustness

Phase: Frontend / Backend Integration & Reliability Engineering
Date: 16 April 2026

## Context / Problem / Challenge

Following the successful integration of forensic analysis improvements, the project faced three interconnected challenges:

**Challenge 1: Frontend Fragmentation**
The results display pipeline involved multiple separate components (GradCAM outputs, distributed UI logic, inconsistent data formatting) that required consolidation into a unified static results interface. The UI needed to display all forensic factors coherently while managing complex interactivity (toggle-able factor details, pointer-following tooltips, responsive layouts).

**Challenge 2: YouTube Download Failures (HTTP 429 Bot-Check)**
Video downloads via yt-dlp were consistently failing with HTTP 429 "Too Many Requests" errors, indicating YouTube was detecting and blocking the requests as automated bot traffic. The initial mitigation strategy—automatically copying the Chrome browser cookie database into the service context—was unreliable and failed silently in headless/service environments with cryptic "Could not copy Chrome cookie database" errors. This created a critical blocker for deployment: users could not analyze YouTube videos without manual intervention.

**Challenge 3: Backend Organization & Scalability**
The backend file structure lacked clear separation between training pipelines and runtime analysis, making it difficult to maintain and extend. Forensic analysis code was dispersed across multiple files without logical grouping.

## Approach / Method

### UI Consolidation Strategy

Rather than maintaining separate output channels, I implemented a unified static results page served by FastAPI that consolidates:
- Video details and thumbnail preview
- Deepfake detection gauge (radial chart with severity-based coloring: low/medium/high)
- Contributing forensic factors list (collapsed/expandable details per factor)
- Timeline chart (vertical bar visualization of forensic metrics)

All this data flows through a single JSON API endpoint from the backend analysis results folder, eliminating the need for separate GradCAM or fragmented output channels.

### Forensic Pipeline Improvements

Four key stabilization improvements were implemented:

1. **Longer Temporal Windows (30 → 120 frames)**
   - Extended history buffers in `WarpingAnalyzer` and `OpticalFlowAnalyzer` classes
   - Reduces frame-to-frame noise; enables z-score-based anomaly detection relative to 120-frame rolling baseline
   - Prevents false positives from natural head motion

2. **Optical Flow Analysis (Farneback Algorithm)**
   - New motion detector class that computes dense optical flow
   - Detects unnatural motion patterns by analyzing z-scores of motion magnitude
   - Blended with landmark-based warping detection for robust motion estimates

3. **Face Region Masking**
   - `extract_face_region()` now uses MediaPipe landmarks (or Haar cascade fallback) to isolate face area
   - Texture, frequency, and chrominance analysis operate on face-only pixels
   - Eliminates background noise and improves artifact specificity

4. **Combined Warping Detection**
   - Merges landmark displacement + optical flow anomaly scores
   - Single unified warping metric balances geometric face deformation with motion irregularities

All 8 forensic factors are preserved and enhanced:
- **Facial Warping** — landmark displacement + optical flow (0-1 score)
- **Lighting Inconsistencies** — frame-to-frame shadow direction variance
- **Texture Mismatches** — sharpness analysis on face region only
- **Temporal Flickering** — inter-frame intensity changes (with frame-size normalization)
- **Eye Alignment Stability** — landmark positional jitter
- **High-frequency Artifacts** — DFT/DCT signature on face region
- **Chrominance Discontinuities** — Cr/Cb channel edge artifacts on face region
- **Compression Artifacts** — Blockiness detection (JPEG/H.264 reencoding)

### YouTube Download Robustness (No Chrome DB Copy)

**Root Cause:**
yt-dlp requires valid browser context (via cookies) to pass YouTube's bot-check. Attempting to auto-copy Chrome's encrypted local cookie database fails in headless/service contexts due to:
- DB file locks (Chrome browser still running/accessing DB)
- OS-level permissions issues (service account vs. user account)
- Encryption/platform-specific issues (Windows: DPAPI; macOS: Keychain)

**Solution Implemented:**

Rather than auto-copying (which silently fails), the server now supports three explicit, deployable cookie strategies:

1. **Environment Variable:**
   ```bash
   export YT_DLP_COOKIES_FILE=/path/to/cookies.txt
   python backend/fastapi_server.py
   ```
   Users pre-export cookies locally: `yt-dlp --cookies-from-browser chrome --cookies cookies.txt --skip-download <URL>`

2. **Per-Request Injection:**
   ```bash
   curl -X POST -H "Content-Type: application/json" \
     -d '{"url":"<VIDEO_URL>","cookies_text":"# Netscape HTTP Cookie File..."}' \
     http://localhost:8000/analyze_url
   ```

3. **API Upload Endpoint:**
   ```bash
   curl -F "file=@cookies.txt" http://localhost:8000/upload_cookies
   ```
   Cookies stored in `backend/downloads/` and automatically picked up by downloader.

All strategies avoid automatic browser DB access; user provides cookies explicitly. Server logic tries cookies in order: request override → environment variable → uploaded file → fails gracefully with actionable error message.

## Response / Solution

### Implementation Details

**Frontend Changes:**
- Replaced multi-channel outputs with unified static `extension/results.html`, `extension/style.css`, `extension/results.js`
- Fixed `renderFactors()` logic: corrected loop structure, moved layout helpers outside item iteration, added null-checks for all DOM components
- Implemented pointer-following tooltips with severity-based glow
- Close Report button calls `/cleanup_results` endpoint before closing tab

**Backend Changes:**
- `forensic_analyser.py`:
  - Added `WarpingAnalyzer` (120-frame rolling buffer, z-score baseline detection)
  - Added `OpticalFlowAnalyzer` (Farneback dense optical flow)
  - `extract_face_region()` uses MediaPipe landmarks with Haar cascade fallback
  - All texture/frequency/chrominance analysis constrained to face region
- `fastapi_server.py`:
  - `download_youtube_video()` accepts optional cookies file override
  - Added retry/backoff logic (configurable via env: `YT_DLP_ATTEMPTS`, `YT_DLP_BACKOFF_SECONDS`)
  - Added user-agent and Accept-Language headers (browser mimicry)
  - Removed automatic Chrome DB copy fallback; explicit logging when skipped
  - New endpoints: `/upload_cookies`, `/analysis_status`, `/cleanup_results`
  - `POST /analyze_url` accepts optional `cookies_text` field

**File Organization:**
- Created `backend/video_analyses/` for runtime analysis modules
- Moved/organized `model_training/` for training artifacts
- Updated imports in `fastapi_server.py` to reference reorganized modules

## Outcome / Reflection

**UI Consolidation Outcome:**
The unified results page successfully displays all forensic data in a single coherent view. The hierarchical factor list with expandable details, combined with the timeline chart, provides users with comprehensive but digestible deepfake detection evidence. JavaScript event handling is now robust with null-checks, preventing silent failures in edge cases.

**Forensic Pipeline Outcome:**
The 4 stabilization improvements significantly reduced false positives while maintaining high sensitivity to genuine artifacts:
- 120-frame windows eliminate noise from natural head motion
- Optical flow + landmark fusion detects complex warping patterns (e.g., asymmetric face reshaping)
- Face region masking reduces background-induced false positives
- Combined approach balances complementary detection strengths

**Download Robustness Outcome:**
By eliminating unreliable automatic Chrome DB copying and providing three explicit cookie strategies, the system is now deployable in diverse environments (local dev, cloud services, CI/CD) without platform-specific DB access puzzles. Users can control cookie lifecycle and receive clear error messages guiding them to resolution.

**Backend Organization Outcome:**
Clear separation between `video_analyses/` (runtime) and `model_training/` (training) improves maintainability and allows future scaling: training can evolve independently of production inference; new analysis modules can be added without cluttering the API layer.
